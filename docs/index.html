Machine Learning is the hot topic. A chatbot is making the hot topic as well. There are several approaches how to build your own chatbot. I want to share how to it recognize a voice on good enough level without any external requests (that means low latency, cheaper, simpler), the same about the text to speech (a digital voice) and how to use local machine learning to improve the discussion. The level of complexity is acceptable even for beginners with machine learning :).

The fact that usability has a big impact on success of web products is a common knowledge. But how do we turn that knowledge into our everyday practice? In a number of cases ‚Äì is the app fast enough or do we know if the button is clickable ‚Äì usability depends on programmer's job. In my talk I will share specific patterns and solutions which we should use in every project to ensure the code is user-centered. My goal is to remind you that all frontend developers are responsible for UX. Sometimes it's really only a few lines of code that can change user's experience completely!

12:20 - 12:50 Micha≈Ç Taszycki - Let's imagine another world

Every day we craft beautiful architectures and use design patterns. Every day we juggle functional, reactive and object-oriented programming. Every day we choose from thousands of frameworks and libraries. We do it because we can! We can let our browsers fetch megabytes of code. We can add another server when the backend can't handle the traffic. We can buy a new phone when the old one get's slow. But let's imagine another world. The world with kilobytes of memory and just one megahertz CPU. The world where the hardware doesn't change, but new programs are better and faster. The world without monads, object-oriented and functional programming. Imagine writing everything from scratch, in a real assembly. For a computer that is not produced for over 25 years. Imagine having so much fun with it, that you can't believe it! What if I tell you such world exists?
